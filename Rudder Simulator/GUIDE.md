# Пошаговое руководство по настройке проекта ESP32S3_NMEA2000_device_simulator

## Обзор проекта
Проект представляет собой симулятор устройств NMEA2000 для платы ESP32-S3, который отправляет данные о температуре и влажности по сети NMEA2000. Проект использует PlatformIO для сборки и управления зависимостями.

## Шаг 1: Установка необходимого ПО
1. **Установите Visual Studio Code** (если еще не установлен)
2. **Установите PlatformIO IDE расширение** в VS Code
3. **Установите Git** для управления зависимостями

## Шаг 2: Клонирование проекта
```bash
git clone https://github.com/ra4hgn/vodoplav.git
cd vodoplav/"Rudder Simulator"
```

## Шаг 3: Настройка PlatformIO
1. Откройте проект в VS Code
2. PlatformIO автоматически определит структуру проекта из файла `platformio.ini`
3. Проверьте конфигурацию в `platformio.ini`:
   - Платформа: `espressif32`
   - Плата: `esp32s3-devkitc-1`
   - Фреймворк: `arduino`

## Шаг 4: Установка библиотек (автоматически через PlatformIO)
PlatformIO автоматически установит все необходимые библиотеки, указанные в `platformio.ini`:
```ini
lib_deps = 
    ttlappalainen/NMEA2000@^4.22.0
    offspring/NMEA2000_esp32@add-esp32s3
    sparkfun/SparkFun BME280@^2.0.9
```

## Шаг 5: Ручная установка библиотек (если требуется)
Если PlatformIO не установит библиотеки автоматически:
1. Откройте PlatformIO Home в VS Code
2. Перейдите во вкладку "Libraries"
3. Поиск и установка:
   - `NMEA2000` от ttlappalainen (версия 4.22.0)
   - `NMEA2000_esp32` от offspring (ветка add-esp32s3)
   - `SparkFun BME280` от sparkfun (версия 2.0.9)

## Шаг 6: Подготовка оборудования
1. Подключите плату ESP32-S3 DevKitC-1 к компьютеру через USB-C
2. Убедитесь, что драйверы USB установлены
3. Определите COM-порт платы (Windows) или /dev/ttyUSB* (Linux/Mac)

## Шаг 7: Компиляция проекта
В VS Code с PlatformIO:
1. Нажмите на иконку PlatformIO на левой панели
2. Выберите "Project Tasks" → "esp32s3" → "Build"
3. Или используйте терминал: `pio run`

## Шаг 8: Загрузка прошивки на устройство
1. Убедитесь, что плата подключена
2. В PlatformIO:
   - "Project Tasks" → "esp32s3" → "Upload"
   - Или терминал: `pio run --target upload`
3. Дождитесь завершения загрузки

## Шаг 9: Мониторинг последовательного порта
Для отладки и просмотра вывода:
1. В PlatformIO:
   - "Project Tasks" → "esp32s3" → "Monitor"
   - Или терминал: `pio device monitor`
2. Настройте скорость: 115200 бод

## Шаг 10: Настройка проекта (опционально)
1. **Изменение PGN сообщений**: В файле `ESP32S3_NMEA2000_device_simulator.ino` измените массив `TransmitMessages[]`
2. **Настройка пинов CAN**: Измените `ESP32_CAN_TX_PIN` и `ESP32_CAN_RX_PIN` если используете другие пины
3. **Включение датчика BME280**: Раскомментируйте строки 136-149 для работы с реальным датчиком

## Шаг 11: Тестирование
1. Подключите плату к сети NMEA2000 через CAN-трансивер
2. Используйте инструменты для мониторинга NMEA2000 (OpenSkipper, Actisense NMEA Reader)
3. Проверьте отправку PGN сообщений:
   - 130316: Температура
   - 130313: Влажность

## Шаг 12: Решение проблем
1. **Ошибки компиляции**: Проверьте установленные библиотеки
2. **Проблемы с загрузкой**: Проверьте подключение платы и драйверы
3. **CAN не работает**: Проверьте подключение CAN-трансивера и настройки пинов

## Важные замечания:
- Проект использует PlatformIO, а не Arduino IDE
- Библиотека `NMEA2000_esp32` требуется специальная версия с поддержкой ESP32-S3
- Для работы с реальным датчиком BME280 раскомментируйте соответствующий код
- По умолчанию проект симулирует данные с помощью потенциометров

## Файлы проекта:
- `platformio.ini` - конфигурация PlatformIO
- `src/ESP32S3_NMEA2000_device_simulator.ino` - основной код прошивки
- `README.md` - краткое описание проекта
